The website [www.humansnot.biz](https://www.humansnot.biz/) and this repository are a small, bottom up, people driven project focused on helping people, regardless of governments, parties, religions, or ideologies.

The goal is simple, at least in theory:

Make it easier for average people to understand a cause, identify who is affected, and figure out how to help in a way that is informed, responsible, and human.

No governments.  
No organizations controlling the narrative.  
No hidden agendas.

Again, **this is a collaborative, open source project.**

---

## What this project is

- A public, collaborative knowledge base about global causes
- Focused on people and real world impact
- Built to reduce confusion, misinformation, and performative activism
- Open to review, critique, and improvement

Each cause tries to answer:
- What is happening (including accountability, because with great power comes great responsibility)
- Who is affected
- How to get and stay informed
- How to act responsibly (time, money, energy)

---

## What this project is not

- A charity or NGO
- A fundraising platform
- A political party or ideological space
- A place for flame wars or social media style debates

Discussion is welcome, but the goal is helping people, not winning arguments. People are actually suffering. Egos can wait.

## How content is created

All content lives in this GitHub repository and is written in plain Markdown files.

The process is intentionally simple:

1. A cause or resource is added or updated  
2. Sources are linked (mandatory)
3. Content is reviewed by others (ideally approved by two people before publishing)  
4. Improvements happen over time  

Nothing here is considered final or absolute.

## GitHub basics (quick guide)

If you are new to GitHub, this is the simplest way to participate (yes, unfortunately you will need to create an account).

### Option 1: Open an issue (easiest)

Use this if you want to report a problem or suggest something.

1. Click the **Issues** tab at the top of this repository  
2. Click **New issue**  
3. Describe what you noticed or want to suggest (it can also be a new cause of course!)
4. Submit  

Thatâ€™s it. For now, requests will be handled manually as best as possible.

---

### Option 2: Edit a page directly (still easy)

Use this if you want to fix or improve text.

1. Navigate to the file you want to edit  
2. Click the pencil icon (Edit)  
3. Make your changes  
4. Scroll down and write a short description  
5. Click **Propose changes**  

GitHub will automatically create a Pull Request (PR).

---

### Option 3: Pull Requests (for advanced users)

If you already know GitHub:
- Fork the repository
- Create a branch
- Make changes
- Open a Pull Request

Please keep PRs focused and explain your reasoning.
